# Project Definition

## 해결할 문제

채용 공고는 여러 검색어와 페이지에 흩어져 있고, 같은 공고를 반복해서 확인해야 합니다. 공고마다 표현 방식이 달라 지원 우선순위를 일관된 기준으로 판단하기도 어렵습니다.

이 프로젝트는 채용 공고의 수집, 중복 제거, 상세 정보 정리, 개인 기준 매칭, 리포트 생성을 하나의 재현 가능한 흐름으로 자동화합니다.

## 대상 사용자

- 여러 개발 직무의 채용 공고를 반복해서 확인하는 개인 구직자
- 검색 결과보다 자신의 위치·경력·기술 조건에 맞는 공고를 먼저 보고 싶은 사용자
- 수집 결과와 추천 이유를 직접 검증하고 싶은 사용자

현재 MVP는 단일 로컬 사용자와 JobKorea 중심으로 설계합니다.

## 목표

1. 같은 공고를 중복 저장하지 않는다.
2. 목록과 상세 페이지의 원본 및 정제 데이터를 함께 남긴다.
3. 매칭 점수와 추천 이유를 설명 가능하게 제공한다.
4. 실행 실패와 데이터 품질을 다시 확인할 수 있게 기록한다.
5. CSV/XLSX 결과를 통해 실제 지원 검토 시간을 줄인다.

## MVP 범위

### 포함

- JobKorea 목록 및 상세 공고 수집
- SQLite 저장과 변경 이력 관리
- 설정 기반 규칙 점수 계산
- 현재 유효한 공고 필터링
- 콘솔, CSV, XLSX 리포트
- 수집 상태 Health Report

### 제외

- 다중 사용자 인증
- 웹 대시보드
- 자동 지원
- LLM 기반 비설명형 점수
- CAPTCHA 및 접근 제한 우회
- 고빈도 병렬 크롤링

## 성공 기준

- 동일 공고 재수집 시 중복 레코드가 생성되지 않는다.
- 파서 fixture와 핵심 점수 규칙이 자동 테스트를 통과한다.
- 실패한 수집 실행은 `crawl_runs`에서 식별 가능하다.
- 리포트에는 점수, 매칭 키워드, 긍정·주의 이유, 원문 URL이 포함된다.
